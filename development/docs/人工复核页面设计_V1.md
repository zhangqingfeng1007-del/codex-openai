# 人工复核页面设计 V1

生成时间：2026-03-25

## 一、页面定位

人工复核页是智能拆解主链路的一部分，不是展示页。它负责承接智能拆解候选结果、展示来源与判定链路、支持人工裁决，并作为正式入库前的最后闸门。

## 二、设计目标

V1 目标只覆盖主链路必需能力：

1. 看智能拆解结果
2. 看来源文件与原文证据
3. 看原始值、标准值和最终复核值
4. 支持接受、修改、驳回
5. 记录人工修改原因
6. 支持后续统一入库

V1 不追求：

1. 全量复杂责任展示
2. 多页面复杂流程
3. 高度自动化审批

## 三、页面核心原则

1. 结果必须可解释
2. 来源必须可追溯
3. 判定链路必须可展开
4. 人工修改必须留痕
5. 页面同时服务业务复核与技术排障

## 四、页面结构

建议采用三栏结构。

### 4.1 左栏：资料与原文

展示内容：

1. 产品基本信息
2. 原始资料包文件列表
3. 当前字段使用的来源文件
4. 原始文件页码定位
5. 原文证据片段
6. 如条件允许，展示原始 PDF/表格预览

目标：

1. 让复核员先确认“值是不是来自正确文件”
2. 验证原始文件、md、切块之间是否一致

### 4.2 中栏：智能拆解候选

展示内容：

1. 标准项名称
2. 原始提取值 `raw_value`
3. 规范化值 `normalized_value`
4. 当前候选状态
5. 置信度
6. 来源类型 `source_type`
7. 抽取方式 `extract_method`

目标：

1. 让复核员看到机器当前判断
2. 区分原文值和规范化值

### 4.3 右栏：人工裁决

支持动作：

1. 接受
2. 修改
3. 驳回
4. 标记待确认
5. 填写复核备注
6. 记录错因

目标：

1. 形成最终候选确认结果
2. 为后续入库提供唯一人工结论

## 五、字段展开视图

每个标准项展开后，必须展示完整判定链路：

1. `source_raw_value`
   原始文件中的来源值
2. `md_text`
   Markdown 中间文本
3. `block_id`
   切块编号
4. `raw_value`
   机器抽取原始值
5. `normalized_value`
   规范化后的标准值
6. `final_value`
   人工确认后的最终值

这条链路用于判断错误发生在哪一层：

1. 原始文件 -> md
2. md -> block
3. block -> 抽取值
4. 抽取值 -> 规范化值

## 六、判定链路展示

页面必须有“判定链路”区块。

展示内容：

1. 来源优先级
   例如：条款 > 产品说明书 > 核保文件 > 费率表
2. 本次实际采用的来源
3. 命中的规则或模型
4. 规范化逻辑
5. 最终入库字段映射

目的：

1. 让业务知道为什么是这个标准值
2. 让技术知道错误发生在哪个步骤
3. 为未来反向生成条款提供可追溯依据

## 七、状态设计

V1 页面建议支持以下状态：

1. `candidate_ready`
2. `review_required`
3. `accepted`
4. `modified`
5. `rejected`
6. `cannot_extract`
7. `cannot_extract_from_clause`

## 八、错因分类

建议复核时支持以下错因标签：

1. 来源文件错
2. 原文定位错
3. 切块错
4. 规则抽取错
5. 标准项映射错
6. 标准值规范化错
7. 多来源优先级错
8. 标准项库不足
9. 资料不足待补

## 九、V1 最小接口

### 9.1 查询任务

`GET /api/v1/clause-parse/tasks/{task_id}`

返回：

1. 候选字段
2. 来源信息
3. 判定链路
4. 当前状态

### 9.2 提交复核

`POST /api/v1/clause-parse/review`

请求体至少包括：

1. `candidate_id`
2. `action`
3. `final_value`
4. `comment`
5. `reason_code`

### 9.3 提交入库

`POST /api/v1/clause-parse/sync`

要求：

1. 只接受复核通过的数据
2. 不允许页面直接写正式表

## 十、V1 开发范围建议

第一批只覆盖：

1. 核心结构字段
2. 单产品复核
3. 单人审核
4. 单次提交

暂不做：

1. 批量审核
2. 复杂责任对比视图
3. 多人协同锁定
4. 高级统计分析

## 十一、与后续版本的关系

V1 的任务是“让主链路可用”。

V2 再考虑增加：

1. 多来源并排对比
2. 差异高亮
3. 审核建议
4. 反馈闭环统计
5. 面向反向生成的风格模板回看
