## 一、信息层级

```
Task
└── Product
    ├── Package
    │   ├── 来源文件：条款 PDF
    │   ├── 来源文件：产品说明书 PDF
    │   ├── 来源文件：费率表 XLSX / 结果费率表
    │   └── 来源文件：核保规则文件（可选）
    ├── Group[]
    │   └── CoverageItem
    │       ├── Source[]         ← 多来源，每个来源一个候选
    │       ├── LogicTrace
    │       └── Decision
    └── DependencyWarning[]
```

说明：

1. `Task` 负责审核上下文和状态
2. `Product` 负责产品快照，不直接等同数据库单表
3. `Package` 负责资料包与来源文件
4. `Group` 负责字段导航与风险聚合
5. `CoverageItem` 负责单标准项裁决
6. `Source` 负责来源证据与候选值

## 二、字段分组体系

### 2.1 分组定义

| group_type | 显示名 | 包含字段 |
|---|---|---|
| `tier_a_basic` | 基础规则字段 | 投保年龄、保险期间、交费期间、交费频率、等待期、宽限期、犹豫期 |
| `tier_a_ci` | 重疾责任字段 | 重疾赔付次数、重疾分组、重疾数量 |
| `tier_a_rate` | 费率类字段 | 投保年龄（费率侧）、交费期间（费率侧）、交费频率（费率侧）（仅在存在费率来源时显示） |
| `tier_b_minor_ci` | 少儿责任字段 | 特定疾病数量、少儿重大疾病数量、少儿特定重大疾病数量 |
| `tier_b_other` | 其他责任字段 | 轻症赔付次数、中症赔付次数（后续扩展） |
| `conflict` | 冲突字段 | 动态分组：多来源值不一致的字段自动归入 |
| `missing` | 缺失字段 | 动态分组：所有来源均无候选的字段 |

**注：** `conflict` 和 `missing` 是动态分组，优先展示，提醒复核员重点处理。

### 2.2 字段状态分布

每个分组标题旁展示本组状态摘要：

```
重疾责任字段（3个字段）
  ✅ 2 已接受   ⚠ 1 待复核   🔴 0 冲突
```

## 四、来源类型定义

| source_type | 含义 | 天数信息可靠性 |
|---|---|---|
| `clause` | 条款 PDF | 最可靠，优先级最高 |
| `product_brochure` | 产品说明书 PDF | 次之，通常与条款一致 |
| `processed_rate` | 处理后费率表 XLSX | 投保年龄仅为0/N周岁占位，无天数，不作为天数来源 |
| `raw_rate` | 原始费率表（含附注） | 偶有公式说明（如月交=年交×0.09），不含投保年龄天数 |
| `underwriting_rule` | 核保规则文件 | 投保年龄天数的第三优先来源 |

**规则**：`source_type=processed_rate` 的投保年龄候选值若为 `0-X周岁`，不得作为最终来源，须从 `clause` 或 `product_brochure` 补充天数说明。

## 五、冲突类型分类

| conflict_type | 说明 | 处理建议 |
|---|---|---|
| `source_value_diff` | 同一字段多来源值不同 | 人工选择采纳哪个来源；条款优先 |
| `format_diff` | 值语义相同但格式不同（如"趸交"vs"一次性付清"） | 系统自动规范化，标记为提示而非错误 |
| `missing_annotation` | 值部分正确但缺少说明（如缺"28天"） | 从次优来源补充说明 |
| `dependency_conflict` | 关联字段之间逻辑矛盾 | 须联动修正（如次数=1但分组=2组） |

## 六、依赖关系图谱

```
重疾责任链
  重疾赔付次数 ──→ 重疾分组
      │               │
      │  次数=1        │  次数>1且无分组声明
      ↓               ↓
   不涉及分组        不分组 / N组

  重疾赔付次数 ──→ 重疾数量
      │
      │  多次赔付时重疾数量通常更多（300+种），单次赔付通常100-120种
      ↓
   数量合理性警告

投保年龄-来源链
  投保年龄(0-X周岁) ──→ 查条款/说明书中的天数说明
      │
      │  found
      ↓
  升级为 0（N天）-X周岁

交费三联
  交费期间 ──→ 交费频率
      │              │
      │  包含趸交     │  趸交时频率无意义
      ↓              ↓
   注意趸交与期间的逻辑关系

少儿责任链（少儿产品才有）
  重疾赔付次数 ──→ 少儿重大疾病数量（子集）
                       │
                       └──→ 少儿特定重大疾病数量（子集）
```

## 七、信息密度控制原则

| 显示层级 | 默认展示 | 按需展开 |
|---|---|---|
| 清单行 | 状态、字段名、候选值摘要（截断）、来源标签 | 完整候选值 |
| 详情-来源 | 来源名称、候选值、置信度 | 原文段落、block原始内容 |
| 详情-链路 | 来源采纳结论、规范化结论 | 完整优先级评估、规范化步骤 |
| 裁决区 | final_value、动作按钮 | 错因下拉（驳回时展开）、备注（始终显示但默认折叠）|

**原则：** 复核员不需要看所有信息才能做出裁决。默认状态应让 80% 的字段在 3 秒内可完成接受。
