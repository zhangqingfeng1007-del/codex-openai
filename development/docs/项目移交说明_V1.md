## 1. 项目目标
本项目当前目标分两条链：

1. 拆解链
- 原始文件 -> parser -> blocks/structured_table -> candidates

2. 匹配链
- candidates -> 标准项定位 -> 标准值精确匹配 -> dry-run 预览

一期目标：
- 所有值人工审核后入库

二期预留：
- 部分高置信度结构化字段允许直接入库

## 5. 当前规则边界
1. 标准值匹配必须字段内字符级精确匹配
2. 不做模糊匹配
3. 不自动创建标准项
4. 不自动新增标准值
5. 当前 review-module 仍是审核/展示工具，不是正式写库入口

## 6. 现阶段不要做的事
1. 不要直接接真实数据库写入
2. 不要继续投入 scan_pdf / OCR 主线
3. 不要为单个噪音样本写强兜底
4. 不要新建重复映射文件，优先复用：
- `coverage_id_mapping.json`
- `coverage_path_mapping.json`
