# 跨设备开发环境方案 V1

生成时间：2026-03-25

## 一、目标

本方案用于支持同一项目在家里和公司两台电脑间无缝切换开发。

核心目标：

1. 两台机器使用相同的开发方案
2. 代码和文档通过 GitHub 同步
3. 长期记忆通过 iCloud 同步
4. 切换设备时不需要重新建立工作方式

## 二、基本原则

1. 两台机器保持同构
2. 代码同步和记忆同步分层处理
3. 不依赖单次对话上下文保存项目状态
4. 关键结论必须落盘

## 三、同步分层

### 3.1 GitHub 同步内容

通过 GitHub 同步：

1. 代码
2. 开发文档
3. 脚本
4. 配置
5. 共享契约文档

不通过 GitHub 同步：

1. 本地虚拟环境
2. 大体量中间产物
3. 本地缓存
4. 私有记忆目录

### 3.2 iCloud 同步内容

通过 iCloud 同步：

1. 长期记忆
2. 项目背景
3. 经验反馈
4. 治理原则
5. 阶段计划

这部分不替代 GitHub，只负责“跨设备连续性”。

## 四、记忆同步方案

### 4.1 Claude Code 的已验证方案

参考目录：

`iCloud Drive/claude-memory/`

示例文件：

1. `MEMORY.md`
2. `feedback_aix_dev_process.md`
3. `feedback_aix_governance.md`
4. `feedback_clause_extraction_scope.md`
5. `feedback_insurance_domain.md`
6. `project_aix_engine.md`

### 4.2 Codex 侧执行原则

Codex 也采用同类方案，但记忆内容独立，不与 Claude Code 混用。

即：

1. 方案一致
2. 目录结构一致
3. 记忆文件各自维护

建议单独使用：

`iCloud Drive/codex-memory/`

## 五、建议目录结构

### 5.1 Git 工作目录

本地仓库目录：

`~/codex-openai`

### 5.2 Codex 记忆目录

建议 iCloud 中维护：

`iCloud Drive/codex-memory/`

建议文件：

1. `MEMORY.md`
2. `project_aix_engine.md`
3. `project_clause_extraction.md`
4. `feedback_aix_governance.md`

### 5.3 本地软链

本地 Codex 使用的记忆目录建议软链到 iCloud 目录，例如：

1. 本地路径：`~/.codex-memory`
2. 实际指向：`~/Library/Mobile Documents/com~apple~CloudDocs/codex-memory`

## 六、新机器初始化步骤

### 第一步：登录账号

1. 登录同一 Apple ID
2. 启用 iCloud Drive
3. 确认 iCloud 中的记忆目录已同步

### 第二步：拉代码

```bash
cd ~
git clone <your-repo-url> codex-openai
cd ~/codex-openai
git pull
```

### 第三步：建立记忆软链

示例：

```bash
ln -s ~/Library/Mobile\\ Documents/com~apple~CloudDocs/codex-memory ~/.codex-memory
```

### 第四步：验证环境

1. `git status`
2. 核对当前分支
3. 确认记忆软链有效
4. 确认关键文档可读

## 七、当前推荐操作方式

### 在任一机器开始工作前

1. `git pull`
2. 检查 iCloud 记忆是否已同步
3. 读取当前阶段计划和关键记忆文件

### 在任一机器结束工作后

1. 重要结论写入文档或记忆文件
2. 代码和文档提交 Git
3. `git push`

## 八、注意事项

1. GitHub 管代码，iCloud 管记忆，不要混用
2. 不要把本地缓存和虚拟环境同步到 GitHub
3. 不要让 Claude 和 Codex 共用同一份记忆文件内容
4. 方法可以统一，记忆必须隔离

## 九、推荐的最小工作习惯

每次换机器时固定做 3 步：

1. `git pull`
2. 检查记忆目录同步
3. 打开当前阶段文档

每次收工时固定做 3 步：

1. 更新关键文档
2. 提交代码与文档
3. `git push`
