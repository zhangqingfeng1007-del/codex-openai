## 一、标准值用途说明

### 在 build_review_task.py 中的使用

1. **候选值验证**：当抽取层 candidate 的 `value` 不在标准值列表中时，标记为 `review_required`（而非 `candidate_ready`）
2. **复核员辅助**：前端补录表单中，`final_value` 输入框下方展示标准值下拉候选
3. **格式规范化**：入库时检查 `final_value` 是否在标准值集合中，不在则需人工确认

### 在前端 app.js 中的使用（后续）

- 补录表单的 `final_value` 下拉列表

## 二、重要提醒：新产品可能产生新标准值

**保险行业持续发展，新产品会出现现有标准值列表中没有的值。**

### 典型新值场景举例

| 字段 | 现有标准值 | 可能出现的新值 |
|---|---|---|
| 重疾赔付次数 | 最多 5次 | 6次、8次（超多次赔付产品）|
| 重疾数量 | 最多 162种 | 200种+（行业正在扩展）|
| 特定重疾数量 | 最多 少儿20种；罕见病20种 | 新的疾病组合分类 |
| 等待期 | 最短 30天 | 0天（无等待期产品）|
| 交费频率 | 最多 趸交，年交，半年交，季交，月交 | 周交、日交（互联网新产品）|
| 转换权 | 5种描述 | 新的转换条件描述 |
| 保单贷款 | 70-90% | 新的贷款比例或条件 |

### 设计要求（必须体现在 build_review_task.py 中）

```python
def validate_against_standard(coverage_name: str, candidate_value: str, standard_values: dict) -> str:
    """
    检查候选值是否在已知标准值列表中。
    返回：
      - "standard"：值在已知标准值集合中，可信度高
      - "new_value"：值不在已知标准值集合中，但格式合法，可能是新标准值
      - "format_error"：值格式明显不符合该字段规范
    """
    known_values = standard_values.get(coverage_name, {}).get('values', [])
    if candidate_value in known_values:
        return "standard"
    # 不阻断，标记为待确认
    return "new_value"
```

当 `validate_against_standard` 返回 `"new_value"` 时：
- item `status` 保持 `review_required`（不降级为 candidate_ready）
- `logic_trace` 中加一条提示：`"候选值不在已知标准值列表中，可能为新标准值，请人工确认"`
- **不阻断**入库流程，只提示

## 四、给 Codex 的具体实现要求

### 4.1 在 build_review_task.py 中集成

新增 `load_standard_values()` 函数：

```python
def load_standard_values() -> dict:
    """
    加载标准值字典，以 coverage_name 为 key。
    来源：coverage_standard_values_v1.json
    返回：{coverage_name: {"coverage_id": ..., "values": [...], "count": N}}
    """
    path = Path(__file__).parent.parent / "data/manifests/coverage_standard_values_v1.json"
    with open(path) as f:
        raw = json.load(f)
    # 展平分组结构为 {coverage_name: info}
    flat = {}
    for group_fields in raw["groups"].values():
        flat.update(group_fields)
    flat.update(raw.get("ungrouped", {}))
    return flat
```

在 `build_coverage_item()` 中调用，影响：
1. `is_standard_value`：布尔字段，加入 item 的 `logic_trace.normalization_trace`
2. 当候选值不在标准值列表时：
   - status 保持 `review_required`（不升为 candidate_ready）
   - logic_trace 加提示：`"候选值'{value}'不在已知标准值列表中，可能为新标准值，请人工确认"`

### 4.2 新标准值扩展机制（文档要求）
