# 并行开发边界与安全协作方案 V1
生效日期：2026-03-28

## 硬规则（不可违反）
1. 文件所有权互斥：同一文件同一时间只有一个开发者负责
2. 分支隔离：Claude 用 `claude/data-fixes`，Codex 用 `codex/dev`，`main` 只接合并
3. 合并前必须交叉检查：提交方→对方 review→用户决定合并
4. `main` 不做日常开发
5. **Codex 禁止接触阿里云数据库**：Codex 不得执行任何连接、查询、写入阿里云数据库的操作。数据库相关任务（gold 导出、产品库查询）由 Claude 或用户人工处理。

## 模块分工

### Claude 负责
- `development/data/gold/*.json`
- `development/data/eval/*.json`（eval 结果，非 whitelist）
- `development/docs/`（评测说明、样本说明）

### Codex 负责
- `development/scripts/extract_tier_a_rules.py`
- `development/scripts/merge_candidates.py`
- `development/scripts/extract_rate_table*.py`
- `development/review-module/`

## 冻结文件（任何人改前必须用户确认）
- `data/manifests/coverage_whitelist_v1.json`
- `data/manifests/sample_manifest.json`
- `data/manifests/coverage_standard_values_v1.json`
- `docs/review_task_json_schema_v2.md`
- `docs/测试边界与答案隔离原则_V1.md`

## 开工前检查（每次）
```bash
git fetch origin
git pull origin main   # 在自己分支同步基线
git status             # 确认工作区干净
```

## 合并流程
1. 开发方在自己分支提交
2. 对方 review（跑 eval 或阅读代码）
3. 用户决定是否合并到 `main`
