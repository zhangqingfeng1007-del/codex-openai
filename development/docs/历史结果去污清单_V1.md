# 历史结果去污清单 V1

制定时间：2026-03-28
制定依据：`拆解开发方法论_V1.md` + `测试边界与答案隔离原则_V1.md`
目的：明确哪些历史评测结果不能再引用为"真实拆解能力"

---

## 说明

本清单列出历史开发过程中，因使用了**答案镜像或手工中间文件**而产生的评测结果。
这些结果属于 **Layer 3 联调阶段**，证明的是系统链路联通性，不证明自动拆解能力。

---

## 一、已污染的历史阶段结果

### 1. 依赖 `product_coverage_db_v1.json` 的阶段（V1~V9 附近）

| 受影响字段 | 表现 | 污染方式 |
|-----------|------|---------|
| 交费期间 | 曾达到高命中率 | 直接从数据库答案镜像读取字段值 |
| 保险期间 | 同上 | 同上 |
| 投保年龄 | 同上 | 同上 |
| 等待期等 | 部分 | 同上 |

**结论**：V1~V9 期间依赖 `product_coverage_db_v1.json` 得出的命中率，**不算真实拆解能力**，不可引用。

---

### 2. 依赖 `product_disease_counts_v1.json` 的阶段

| 受影响字段 | 表现 | 污染方式 |
|-----------|------|---------|
| 重疾数量 | 曾快速达到 100% | 直接从已统计病种数量结果读取 |

**结论**：该阶段重疾数量的命中率**不算真实拆解能力**。
当前 V17 重疾数量 100% 已改为从条款附录序号自动计数，为**真实能力**。

---

### 3. `rate_table_candidates_v1.json`（手工维护文件）

| 状态 | 说明 |
|------|------|
| 文件性质 | 手工整理的费率表结构化结果，非系统自动生成 |
| 当前影响 | 存在于 `merge_candidates.py` 中，但**当前未为任何产品贡献实际输出** |
| 历史影响 | 不确定，可能在早期阶段补充了部分产品的交费期间 |
| 处理方式 | 保留过渡期使用，待费率表 PDF 提取器完成后替换并重验 |

## 三、结果引用规范

今后引用评测结果时，必须标注：

1. **基线版本**（如 V17）
2. **产品范围**（如 8产品）
3. **来源类型**：真实拆解能力 / 联调验证结果

**禁止**：引用历史 V9 及以前的命中率作为系统真实能力证明。
