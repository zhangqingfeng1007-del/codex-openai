# 共享契约清单 V1

生成时间：2026-03-25

## 一、目的

本清单用于定义拆解项目与保险智能体主项目之间必须共享且统一维护的对象。共享契约只能有一份正式口径，不允许两边各自演化。

## 二、契约分层

共享契约分为五类：

1. 标准项契约
2. 字段映射契约
3. 枚举值契约
4. 入库结构契约
5. 来源追溯契约

## 三、标准项契约

统一对象：

1. `cmb_coverage`
2. `coverage_id`
3. `coverage_name`
4. 标准项白名单
5. 标准项禁用范围

要求：

1. 拆解项目和主项目必须使用同一份标准项定义
2. “招行”前缀字段是否有效，只能有一个正式判断
3. 标准项增删改必须同步通知两边

## 四、字段映射契约

统一对象：

1. 中文字段名
2. 内部字段名
3. DB 字段名
4. 标准值落库字段

当前典型对象：

1. `投保年龄`
2. `保险期间`
3. `交费期间`
4. `交费频率`
5. `等待期`
6. `重疾赔付次数`
7. `重疾分组`
8. 费率表字段映射

要求：

1. 不允许拆解项目和主项目各有一套字段名翻译
2. 所有映射必须以共享配置或共享文档为准

## 五、枚举值契约

统一对象：

1. 性别枚举
2. 社保枚举
3. 吸烟状态枚举
4. 保费状态枚举
5. 交费方式枚举
6. 期间类型枚举

当前示例：

1. `gender: 0=不限, 10=女, 11=男`
2. `pay_type: 0=趸交, 1=月交, 3=季交, 6=半年交, 12=年交`

要求：

1. 枚举值只能有一套正式定义
2. 评分、检索、拆解、入库都必须引用同一份定义

## 六、入库结构契约

统一对象：

1. `cmb_product_coverage`
2. `cmb_product_rate`
3. `cmb_product_disease`
4. 其他后续标准库表

要求：

1. 拆解项目最终输出必须对齐正式库结构
2. 主项目只能消费正式库或正式入库结构
3. 不允许主项目直接消费中间 JSON、blocks 或 Markdown

## 七、来源追溯契约

统一对象：

1. `source_type`
2. `source_file`
3. `evidence`
4. `extract_method`
5. `status`

要求：

1. 所有重要入库信息必须可追溯
2. 条款、费率、说明书等来源需显式记录
3. 无来源数据不得入库

## 八、状态契约

统一对象：

1. `candidate_ready`
2. `review_required`
3. `cannot_extract_from_clause`
4. `degraded`
5. `review_passed`

要求：

1. 状态含义在两个项目间必须一致
2. 主项目不应自定义另一套拆解状态语义

## 九、当前优先级最高的共享契约

优先落地三类：

1. 标准项白名单与 coverage 映射
2. 费率字段映射与枚举字典
3. 标准入库输出结构

## 十、维护原则

1. 共享契约变更必须同步更新文档和代码
2. 共享契约优先沉淀为配置，不写死在多个脚本中
3. 若业务语义不清，由业务负责人确认后再改动
