# 金标准导出规范 V1

## 1. 目的

本规范用于指导有临时数据库访问权限的同事，导出 10 款重疾产品的标准拆解结果，形成条款智能拆解一期的“金标准结果集”。

本次导出的结果不是为了直接写回正式库，而是用于：

1. 建立开发与评测基线
2. 与 Markdown 中间文件做字段级对齐
3. 为规则抽取、LLM 映射、人机复核提供对照标尺

## 2. 适用范围

本次仅适用于桌面 `10款重疾` 样本集中的 10 款产品。

产品 ID 清单：

1. `1134`
2. `1464`
3. `1548A`
4. `1568`
5. `1578`
6. `1803`
7. `851A`
8. `864`
9. `889`
10. `919A`

## 3. 导出原则

1. 导出结果必须来自统一临时数据库中的真实标准拆解结果。
2. 不允许人工凭记忆补录结果替代数据库导出。
3. 如果同一产品在库中存在多版本或多来源记录，必须先标记冲突，不能自行拍板选一个。
4. 本次导出只保留当前有效标准项，不纳入“招行”前缀无效字段。
5. 导出文件只作为金标准样本使用，不构成生产正式结论。

每条金标准记录至少包含以下字段：

1. `product_id`
2. `product_name`
3. `coverage_id`
4. `coverage_name`
5. `standard_content`
6. `is_optional_coverage`
7. `source_table`
8. `exported_at`
9. `exported_by`

## 6. 建议 SQL 导出模板

以下 SQL 为导出模板，需要由有数据库权限的同事根据实际库名、表名、字段名校正后执行。

### 6.1 单产品导出模板

```sql
SELECT
    p.product_id,
    p.product_name,
    pc.coverage_id,
    c.coverage_name,
    pc.standard_content,
    pc.is_optional_coverage
FROM cmb_product_coverage pc
LEFT JOIN cmb_coverage c
    ON pc.coverage_id = c.coverage_id
LEFT JOIN cmb_product p
    ON p.product_id COLLATE utf8mb4_unicode_ci = pc.product_id
WHERE pc.is_deleted = 0
  AND c.is_deleted = 0
  AND p.is_deleted = 0
  AND pc.product_id = '1134'
  AND c.coverage_name NOT LIKE '招行%'
ORDER BY pc.coverage_id;
```

### 6.2 批量导出模板

```sql
SELECT
    p.product_id,
    p.product_name,
    pc.coverage_id,
    c.coverage_name,
    pc.standard_content,
    pc.is_optional_coverage
FROM cmb_product_coverage pc
LEFT JOIN cmb_coverage c
    ON pc.coverage_id = c.coverage_id
LEFT JOIN cmb_product p
    ON p.product_id COLLATE utf8mb4_unicode_ci = pc.product_id
WHERE pc.is_deleted = 0
  AND c.is_deleted = 0
  AND p.is_deleted = 0
  AND pc.product_id IN ('1134','1464','1548A','1568','1578','1803','851A','864','889','919A')
  AND c.coverage_name NOT LIKE '招行%'
ORDER BY pc.product_id, pc.coverage_id;
```

说明：

1. 上述 `COLLATE utf8mb4_unicode_ci` 来自你们现有文档口径，用于解决 `cmb_product` 与 `cmb_product_coverage` 的字符集排序规则差异。
2. 如果临时库的表名或删除标记字段不同，以实际库结构为准。
3. 如果标准值不在 `cmb_product_coverage.standard_content`，而在其他责任明细表，则必须补充来源说明。

## 8. 人工核对要求

导出后必须做一次人工核对，重点检查：

1. `product_id` 与样本清单一致
2. `product_name` 与原始文件对应正确
3. 是否误带“招行”前缀无效字段
4. 是否存在 `coverage_name` 为空
5. 是否存在 `standard_content` 为空但理论上应有值
6. 是否存在重复记录、冲突记录、异常记录

如果发现冲突，处理原则如下：

1. 先记录冲突
2. 不自行删除
3. 由业务/拆解同事确认后再形成正式金标准文件
