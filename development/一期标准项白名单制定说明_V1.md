# 一期标准项白名单制定说明 V1

## 1. 目标

在条款智能拆解一期中，不追求覆盖全部标准项，而是先明确一批高频、稳定、可评测、可复核的标准项白名单，作为一期正式开发与验收范围。

白名单的作用：

1. 固定一期开发边界
2. 固定一期评测口径
3. 固定规则开发优先级
4. 防止模型和规则在全量标准项上无序扩散

## 2. 制定原则

1. 一期只纳入重疾险场景下真实高频字段。
2. 优先纳入金标准样本中稳定出现、便于核对的字段。
3. 规则可稳定处理的字段优先进入一期。
4. 复杂责任字段可以进入一期，但默认走 `rule+llm` 或人工复核，不进入全自动。
5. 当前无效使用范围的“招行”前缀字段一律不纳入。
6. 白名单一旦确定，即作为一期开发、测试、导出、验收统一口径。

## 3. 白名单分层

建议分为三层，不混为一类。

### 3.1 A类：一期核心字段

定义：

1. 高频
2. 结构相对稳定
3. 可优先规则化
4. 必须进入一期验收

建议字段：

1. 投保年龄
2. 保险期间
3. 交费期间
4. 等待期
5. 重疾赔付次数
6. 中症赔付次数
7. 轻症赔付次数

### 3.2 B类：一期扩展字段

定义：

1. 业务价值高
2. 结构半稳定
3. 需要 `rule+llm` 配合
4. 进入一期，但默认人工复核

建议字段：

1. 身故责任
2. 豁免责任
3. 少儿特疾责任
4. 成人特定疾病责任
5. 首次赔付比例
6. 后续赔付比例

### 3.3 C类：一期观察字段

定义：

1. 价值较高
2. 文本复杂
3. 一期先纳入观察，不强行承诺自动准确率

建议字段：

1. 可选责任
2. 重疾分组情况
3. 保额限制

## 4. 判定标准

某个标准项是否纳入一期白名单，建议按以下四项判断：

1. 在 10 款金标准样本中是否稳定出现
2. 是否能在 Markdown 中定位到明确证据
3. 是否能用当前标准值口径表达
4. 是否适合作为真实业务复核对象

如果其中任意两项明显不满足，则建议不进入一期正式白名单。

## 5. 字段抽取方式建议

建议在白名单文件中同步标注字段抽取方式：

1. `[rule]`
2. `[rule+llm]`
3. `[llm]`

一期建议如下：

1. 投保年龄 `[rule]`
2. 保险期间 `[rule]`

4. 等待期 `[rule]`
5. 重疾赔付次数 `[rule]`
6. 中症赔付次数 `[rule]`
7. 轻症赔付次数 `[rule]`
8. 身故责任 `[rule+llm]`

## 7. 产出流程

建议按以下顺序制定白名单：

1. 汇总 10 款金标准中的全部标准项
2. 统计每个标准项出现次数
3. 与 Markdown 抽样对照，确认是否可定位
4. 业务同事确认是否属于当前有效范围
5. 最终形成 `coverage_whitelist_v1.json`

## 8. 验收标准

白名单确定后，必须满足：

1. 有明确字段清单
2. 有明确分层
3. 有明确抽取方式
4. 有明确一期是否必做
5. 与文档、实现、测试、导出口径一致
