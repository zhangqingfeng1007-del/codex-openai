# Markdown文本块切分设计 V1

## 1. 目标

将 `doc_to_md` 输出的条款 Markdown 切分为结构化文本块，供后续规则抽取和 LLM 映射使用。

本模块只负责把 Markdown 转为稳定、可追溯、可定位的文本块，不负责标准项判断，也不负责标准值定稿。

## 2. 输入

输入文件路径：

`md_cache/{product_id}_clause_pdf.md`

例如：

`md_cache/1134_clause_pdf.md`

## 3. 输出结构

每个文本块输出以下字段：

1. `block_id`
2. `page`
3. `block_type`
4. `title_path`
5. `text`
6. `char_count`

建议结构：

```json
{
  "block_id": "1134_p4_b12",
  "page": 4,
  "block_type": "paragraph",
  "title_path": ["2.我们保什么", "2.2 投保条件"],
  "text": "本合同接受的投保年龄为出生满28天至50周岁。",
  "char_count": 24
}
```

### 3.1 字段定义

1. `block_id`
   格式：`{product_id}_p{页码}_b{块序号}`
2. `page`
   页码，从 Markdown 中提取
3. `block_type`
   取值：`paragraph / table / title / list`
4. `title_path`
   当前块所属章节路径，数组
5. `text`
   原始文本
6. `char_count`
   文本字符数

## 4. 切分规则

### 4.1 标题识别

标题识别规则：

1. `#` 识别为一级标题
2. `##` 识别为二级标题
3. `###` 识别为三级标题

如果 Markdown 中标题层级缺失或不规范，保留原始顺序，不自行猜测业务层级。

### 4.2 表格识别

满足 Markdown 表格语法的连续行识别为 `table`：

1. 含表头行
2. 含分隔线行，如 `| --- | --- |`
3. 后续连续表格内容行归入同一表格块

### 4.3 列表识别

以下形式识别为 `list`：

1. `- ` 开头的无序列表
2. `1.` `2.` `3.` 开头的有序列表

连续同类列表项可合并为一个列表块。

### 4.4 段落识别

除标题、表格、列表之外的连续文本行，识别为 `paragraph`。

### 4.5 空行规则

空行作为段落分隔符。

即：

1. 连续非空文本行为同一候选段落
2. 遇空行则结束当前段落块

## 5. title_path 构建规则

`title_path` 用于表达当前文本块的章节归属。

### 5.1 构建方式

1. 维护一个当前标题层级堆栈
2. 遇到一级标题时，重置一级以下堆栈
3. 遇到二级标题时，保留一级，替换二级及以下
4. 遇到三级标题时，保留一级、二级，替换三级及以下

### 5.2 继承规则

每个非标题块都继承其出现位置对应的标题链路。

例如：

```json
["2.我们保什么", "2.2 投保条件"]
```

### 5.3 标题块本身

标题块自身也输出为一个 `title` 块，其 `title_path` 为当前标题生效后的完整路径。

## 6. Tier A 字段锚点定位建议

以下锚点仅用于后续规则召回，不直接作为正式抽取结果。

### 6.1 投保年龄

关键词或模式：

1. `投保年龄`
2. `年龄范围`
3. `出生满`
4. `周岁`
5. 正则示例：`投保年龄|出生满\\d+天|\\d+周岁`

### 6.2 保险期间

关键词或模式：

1. `保险期间`
2. `保障期间`
3. `保至`
4. `终身`
5. 正则示例：`保险期间|保障期间|保至.*岁|终身`

### 6.3 交费期间

关键词或模式：

1. `交费期间`
2. `缴费期间`
3. `交费年期`
4. `年交`
5. 正则示例：`交费期间|缴费期间|\\d+年交|交至\\d+岁`

### 6.4 等待期

关键词或模式：

1. `等待期`
2. `自本合同生效之日起`
3. `180天`
4. `90天`
5. 正则示例：`等待期|\\d+天等待期|生效之日起.*\\d+天`

### 6.5 宽限期

关键词或模式：

1. `宽限期`
2. `宽限`
3. `自保险费到期日起`
4. `60日`
5. 正则示例：`宽限期|宽限|\\d+日内`

### 6.6 犹豫期

关键词或模式：

1. `犹豫期`
2. `签收本合同之日起`
3. `解除合同`
4. `退还`
5. 正则示例：`犹豫期|签收.*日起.*\\d+日内|解除合同`

### 6.7 重疾赔付次数

关键词或模式：

1. `重大疾病保险金`
2. `重疾赔付次数`
3. `给付次数`
4. `赔付次数`
5. 正则示例：`重疾赔付次数|重大疾病.*给付.*次|赔付.*\\d+次`

### 6.8 重疾分组

关键词或模式：

1. `分组`
2. `重大疾病分组`
3. `同组`
4. `不同组`
5. 正则示例：`分组|重大疾病分组|同组|不同组`

### 6.9 重疾数量

关键词或模式：

1. `重大疾病`
2. `种重大疾病`
3. `疾病种类`
4. `保障.*种`
5. 正则示例：`\\d+种重大疾病|疾病种类|保障.*\\d+种`

## 7. 输出格式

输出为 JSON 数组。

存放路径：

`data/blocks/{product_id}_blocks.json`

例如：

`data/blocks/1134_blocks.json`

## 8. 一期不处理的情况

以下情况一期不做自动化深处理：

1. 图片内文字
2. 表格嵌套
3. 跨页合并段落

对于跨页合并段落，标记为：

`cross_page_candidate`

并转人工复核。
