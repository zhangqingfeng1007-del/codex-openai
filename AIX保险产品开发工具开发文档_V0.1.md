# AIX保险产品开发工具开发文档

**版本：** V0.1  
**文档状态：** 需求与开发初稿，待产品确认  
**定位：** 独立保险产品开发工具，不是面向终端用户的推荐页，也不是智能体附属试算页  
**依据来源：** `保险产品定价20200923.xlsm` 工作簿结构与现有测试原型分析  
**更新日期：** 2026-03-21

---

## 1. 文档目标

本工具目标不是单纯给某个现有产品打分，而是提供一套可配置、可试算、可输出保费矩阵的**保险产品开发工具**，供产品经理、精算、研发在后台完成产品责任设计、病种配置、发生率映射、定价参数配置与保费计算。

后续智能体接入方式不是“替代该工具”，而是“调用该工具的结构化能力”。

---

## 2. 当前结论

基于 `保险产品定价20200923.xlsm` 的分析，当前可以明确：

1. 该 Excel 不是单一费率表，而是一套完整的产品定价工作簿。
2. 工作簿已经具备“主输入页 + 责任定义页 + 病种选择页 + 发生率页 + 现值计算页 + 保费结果页”的完整结构。
3. 若要做成正式工具，必须围绕以下三类核心能力建设：
   - 产品责任设计
   - 病种与发生率映射
   - 保费矩阵计算与结果输出
4. 当前测试原型已验证：真实死亡率表、部分真实疾病发生率表、部分比率曲线可被导入并参与试算。
5. 但当前原型仍不等于 Excel 全逻辑复刻，尤其是病种选择驱动的完整精算链路尚未完全实现。

## 3. Excel 逻辑梳理

  作用：统一录入产品建模点、责任开关、保额、保障期、缴费期、定价假设、准备金假设、利润测试假设。

  作用：责任给付表达式及标准给付结构表。
- `Annuity_Table`
  作用：年金给付相关表。
- `EcoTable`
  作用：经济假设表。
- `SurrenderTable`
  作用：退保率表。
- `CommissionRateTable`
  作用：佣金率表。
- `AMR_Table`
  作用：医疗相关发生率表。

### 3.1.3 病种发生率与病种模板页

- `CI_Male` / `CI_Female`
  作用：重疾发生率基础表。
- `MinorCI_Male` / `MinorCI_Female`
  作用：轻症发生率基础表。
- `ModerateCI_Male` / `ModerateCI_Female`
  作用：中症发生率基础表。
- `CI_MaleChoosen` / `CI_FemaleChoosen`
  作用：重疾病种选择模板与汇总结果。
- `MinorCI_MaleChoosen` / `MinorCI_FemaleChoosen`
  作用：轻症病种选择模板与汇总结果。
- `ModerateCI_MaleChoosen` / `ModerateCI_FemaleChoosen`
  作用：中症病种选择模板与汇总结果。
- `SpecialCI_MaleChoosen` / `SpecialCI_FemaleChoosen`
  作用：特疾病种模板与汇总结果。
- `OtherDisease_MaleChoosen` / `OtherDisease_FemaleChoosen`
  作用：其他疾病病种模板。
- `OtherDisease_MaleAcc` / `OtherDisease_FemaleAcc`
  作用：其他疾病累计发生率结果。

### 3.1.4 责任现值计算页

- `CalculationOfBE_CI`
- `CalculationOfBE_MinorCI`
- `CalculationOfBE_ModerateCI`
- `CalculationOfBE_SpecialCI`
- `CalculationOfBE_OtherDisease`

这些页的共同作用是：

1. 列出病种清单
2. 配置是否选择
3. 配置给付比例
4. 配置自动附加调整与个别加载
5. 按年龄输出男女汇总发生率

### 3.1.5 保费结果与测试页

- `Premium`
  作用：单一参数组合下的保费计算主表。
- `PremiumTable`
  作用：按年龄、性别、保障期、缴费期展开结果矩阵。
- `Prem_WOP_Test`
  作用：豁免责任测试比率。
- `Prem_MultipleCI_Test`
  作用：多次重疾责任测试比率。
- `MultipleCI_Markov`
  作用：多次重疾状态转移与 Markov 假设表。

## 3.3 CalculationOfBE 页的关键逻辑

## 3.3.1 重疾页 `CalculationOfBE_CI`

已识别逻辑：

1. 病种逐项列出。
2. 每一行包含：
   - 病种名称
   - 是否选中
   - 给付比例 `Benefit %`
   - 自动加载 `automat. Loading`
   - 个别加载 `indiv. Loading`
   - 截止年龄 `To Age`
3. 右侧按年龄输出：
   - `Male`
   - `Female`
4. 右侧通过 `VLOOKUP(..., CIMaleTotal / CIFemaleTotal, 3, FALSE)` 取值。

结论：
该页不是纯展示页，而是**病种选择与发生率汇总控制页**。

## 3.3.2 轻症页 `CalculationOfBE_MinorCI`

已识别逻辑：

1. 轻症支持默认包与扩展病种并存。
2. `30种非主要轻症` 带有个别调整公式。
3. 男女性别分别通过 `MinorCIMaleTotal` / `MinorCIFemaleTotal` 汇总。

结论：
轻症不能只作为一个固定因子，必须支持病种层级配置。

## 3.3.3 中症页 `CalculationOfBE_ModerateCI`

已识别逻辑：

1. 中症默认存在一组预选病种。
2. 每个病种有给付比例字段。
3. 男女性别分别通过 `ModerateCIMaleTotal` / `ModerateCIFemaleTotal` 汇总。

结论：
中症必须单独配置，不应与重疾或轻症混用固定比例代替。

## 3.3.4 特疾页 `CalculationOfBE_SpecialCI`

已识别逻辑：

1. 特疾页可复用部分轻症病种池。
2. 默认模板中只选中特定包，例如 `30种非主要轻症`。
3. 男女性别分别通过 `SpecialCIMaleTotal` / `SpecialCIFemaleTotal` 汇总。

结论：
特疾需要作为单独责任能力建设，而不是简单并入轻症。

## 3.3.5 其他疾病页 `CalculationOfBE_OtherDisease`

已识别逻辑：

1. 以“恶性肿瘤、心梗、脑中风后遗症、肾衰竭”等为核心病种。
2. 右侧不是单表 `VLOOKUP`，而是按 `OtherDisease_*Choosen` 的行区间做 `SUM`。
3. 说明该页是组合式病种累计结果，而不是单字段映射。

结论：
其他疾病需要独立建模，不应直接混为重疾。

## 4. 对产品开发工具的重新定位

本工具应定位为：

**保险产品开发与定价配置工具**

核心使用者：

- 产品经理
- 精算人员
- 保险责任设计人员
- 后端配置运营人员

不直接面向终端消费者。

工具核心输出：

1. 产品责任结构
2. 病种配置方案
3. 发生率映射结果
4. 定价参数方案
5. 各年龄、性别、保障期、缴费期下的保费矩阵

智能体后续接入方式：

1. 智能体读取产品工具的结构化结果
2. 智能体不直接做底层病种定价
3. 智能体负责解释、推荐、追问和输出

## 11. 必须确认的问题

以下问题当前不能靠臆想拍板，需产品确认后再实现：

1. 第一阶段是否只做重疾险产品开发工具，还是同时覆盖医疗、寿险、年金基础能力？
2. 病种选择页是否要允许“逐病种编辑”，还是先支持“模板包 + 个别调整”？
3. `CalculationOfBE_CI`、`CalculationOfBE_ModerateCI`、`CalculationOfBE_SpecialCI` 是否都纳入第一阶段完整页面？
4. `OtherDisease` 第一阶段是否纳入正式产品设计链路，还是先作为预留能力？
5. 多次重疾的 Markov 逻辑第一阶段要不要完整实现，还是先保留比率法近似？
6. 豁免责任第一阶段是否按 Excel 测试比率近似，还是要求完整公式化？
7. 第一阶段输出是否要包含现金价值、准备金、利润测试，还是先只输出保费矩阵？
8. 产品开发工具的主使用对象是内部产品/精算团队，还是未来也要给 B 端客户使用？

## 12. 第一阶段开发建议

建议按以下顺序推进：

1. 固化产品开发工具定位
2. 先实现病种选择与发生率映射
3. 再实现重疾/中症/轻症/特疾完整试算
4. 再补多次重疾与豁免
5. 最后补现金价值、利润测试、准备金输出

原因：

- 当前最大风险不在页面，而在病种与发生率映射是否准确
- 如果这一步没先打稳，后面的保费矩阵和智能体接入都会失真

---

## 13. 当前结论

当前最合适的产品方向是：

**先开发独立的保险产品开发工具，再把其结构化能力接入 AIX 智能体。**

该工具的第一阶段核心不是“产品推荐”，而是：

- 病种选择
- 发生率映射
- 责任设计
- 保费矩阵试算

只有这四个核心打稳，后续接入智能体后，智能体输出的推荐、解释、评分才会真正可靠。
