# 重疾险产品定价工具 — 精算核对清单

> 版本：v1.2（2026-03-21）
> 用途：供精算部门同事逐项核对，结论填入最右列
> 工具路径：`insurance-product-tool.html`（双击浏览器打开）
> 数据路径：`actuarial-data.js`（与主工具同目录）

---

## 使用说明

1. 打开 `insurance-product-tool.html`，进行各项配置后点击"计算公允保费"
2. 结合 `actuarial-data.js` 和原始 Excel 底稿（`保险产品定价20200923.xlsm`）逐项对照
3. 每项填写：`✅ 通过` / `❌ 有偏差（附说明）` / `⚠️ 待确认`
4. 对有偏差的项目，填写具体偏差说明，交 Claude Code 修复后重新核对

## B. 精算口径核对

### B1 多脱退模型结构

| # | 核对项 | 验证方法 | 结论 |
|---|-------|---------|------|
| B1.1 | 死亡参与 px 递减（合同终止） | 代码：`rawCIRate` 计算；`px *= max(0, 1 - qd - rawCIRate - lapseRate)` | |
| B1.2 | 重疾参与 px 递减（合同终止） | 同上，rawCIRate 含重疾发生率 | |
| B1.3 | 轻症**不**参与 px 递减（合同继续） | 代码验证：`qmCIT` 不在 `rawCIRate` 中，只在 `benefitAmt` 中 | |
| B1.4 | 中症**不**参与 px 递减（合同继续） | 同上 | |
| B1.5 | 特定疾病**不**参与 px 递减（合同继续） | 同上，`qSCIT` 仅在 `benefitAmt` 中 | |
| B1.6 | 退保率作为额外脱退分量（与死亡+重疾并列，不是修正系数） | 代码：`lapseRate` 在 `px *= max(0, 1 - qd - rawCI - lapseRate)` 中 | |

### B2 重疾聚合/自定义互斥

| # | 核对项 | 验证方法 | 结论 |
|---|-------|---------|------|
| B2.1 | 聚合模式下，rawCIRate = CI_AGG，不叠加逐病种 | 代码：`ciMode==='aggregate'` 分支直接取 `CI_AGG[gender][att]` | |
| B2.2 | 自定义模式下，`isAggregate:true` 的条目被过滤 | 代码：`computeCI()` 中 `if(!d.chosen||d.isAggregate)continue` | |
| B2.3 | 自定义模式的 px 递减中，聚合项同样被过滤 | 代码：rawCIRate IIFE 中 `if(!d.chosen||d.isAggregate)continue` | |
| B2.4 | 功能验证：自定义仅选恶性肿瘤，GP 应远小于聚合模式 GP | 在工具中操作验证 | |

### B3 ToAge 口径

| # | 核对项 | 验证方法 | 结论 |
|---|-------|---------|------|
| B3.1 | 赔付计算中：`att >= toAge` 时该病种发生率为 0 | 代码：`computeCI()`、`computeAggregate()` 的 `if(att>=toAge)continue` | |
| B3.2 | 生存概率递减中：同样尊重 toAge（`att>=(d.toAge||105)continue`） | 代码：rawCIRate IIFE 中 `if(att>=(d.toAge||105))continue` | |
| B3.3 | 两处口径一致：不会出现"赔付截断但仍影响 px"的情况 | 代码一致性确认 | |
| B3.4 | 功能验证：将某病种 toAge 设为 60，计算时 60 岁后该病种贡献为 0 | 在工具中操作验证 | |

### B4 多缴费方式与趸交口径

| # | 核对项 | 验证方法 | 结论 |
|---|-------|---------|------|
| B4.1 | 趸交（值=0）在计算层统一映射为 PPP=1 | 代码：`resolvePayPeriod(0, T) === 1` | |
| B4.2 | 三处调用（calculate/generateRateTable/sensitivityAnalysis）均使用同一 helper | 代码：全局搜索 `resolvePayPeriod` | |
| B4.3 | premPayPeriod 在 getParams() 中的静默回退（无选时=20）已被 calculate() 守卫拦截 | 代码：`if(!p.payOptions.length)` 早返回 | |
| B4.4 | 功能验证：5年交 GP > 10年交 GP > 20年交 GP（同其他参数） | 在工具中操作验证 | |

### B5 保额口径

| # | 核对项 | 验证方法 | 结论 |
|---|-------|---------|------|
| B5.1 | 重疾责任使用 `ciSumAssured`（独立配置） | 代码：`ciSumAssured*qCIT` | |
