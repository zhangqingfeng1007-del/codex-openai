# AIX 一期正式开发范围与实施计划 V1

## 一期目标

在第一阶段做成可正式内部试运行的 `重疾险保险智能体`，打通最小可用商用链路：

`对话输入 -> 需求分析 -> Top3 推荐 -> 卡片展示 -> 精算性价比说明`

支持输入方式：

- 文字
- 语音
- 图片

一期只面向：

- 个人用户
- 重疾险 `6001`

## 一期交付范围

必须交付：

### 1. 智能体主链路

- 多轮对话
- 需求收集
- `generate_needs_report`
- Top3 推荐结果
- 推荐卡片展示

### 2. 输入能力

- 文字输入
- 语音输入
- 图片上传
- 图片缩略图预览正常显示

### 3. 评分与精算

- 7维评分可运行
- 精算性价比可显示
- 精算失败时明确降级，不影响其他维度

### 4. 前端页面

- 桌面网页可用
- 手机网页可用
- SSE 流式响应稳定
- 推荐卡片、需求报告卡可正常渲染

### 5. 管理后台最小能力

- 需求报告统计
- 问题上报查看
- 基础精算配置入口或占位

### 6. 测试与移交

- 主链路验收通过
- 页面关键交互验收通过
- 可移交给精算部门核对

## 一期明确不做

以下内容不纳入一期正式交付：

- 百万医疗险正式接入
- 寿险、年金险接入
- Markov 多次赔付模型
- WOP 独立精算建模
- 现金价值、准备金、返还型责任
- 全量动态假设管理平台
- B端企业版正式能力

这些只允许做：

- 设计预留
- 文档说明
- 占位字段

不能作为一期完成项对外表述。

## 开发主线

### 主线 A：重疾险智能体正式链路

- 前端聊天页
- BFF 会话 / SSE / ASR 代理
- Python AI 编排
- 推荐与评分
- Java 精算调用
- 前端卡片展示

### 主线 B：产品定价工具精算核对版

- 独立定价工具继续保留
- 供精算部门核对责任口径和费率逻辑
- 作为智能体精算能力的辅助验证工具

## 实施顺序

### 阶段 1：收口当前版本，3~5 天

目标：把现有链路做稳。

完成项：

- 修复图片预览 bug
- 图片前端压缩
- Sprint 4 剩余项完成
- 管理后台“需求报告统计”Tab
- 产品卡性价比徽章占位
- 主链路测试补齐
- 精算移交材料整理

### 阶段 2：重疾险一期联调验收，3~5 天

目标：形成可内部试运行版本。

完成项：

- 前后端联调
- SSE / 语音 / 图片链路回归
- Top3 推荐结果核对
- 精算评分降级校验
- 精算部门核对
- 修复验收问题

### 阶段 3：下一阶段启动准备，2~3 天

目标：为二期扩展做准备，不急着开发。

输出项：

- 百万医疗险扩展方案
- 图片分析深化方案
- 动态假设管理方案
- 下一阶段排期建议

## 角色分工建议

- 前端：聊天页、输入栏、卡片、后台统计页
- BFF：SSE 代理、ASR token、会话接口、后台接口
- Python AI：对话编排、需求分析、评分、图片理解
- Java 精算：公允定价、错误码、配置读取
- 精算部门：责任口径、假设口径、费率合理性核对
- Codex：代码审查、测试补漏、治理把关

## 一期验收标准

一期通过必须同时满足：

### 1. 用户能完成一次完整咨询

- 输入需求
- 补充信息
- 生成需求报告
- 得到 Top3 推荐

### 2. 推荐结果可解释

- 有评分
- 有推荐理由
- 精算可用时显示性价比
- 精算不可用时明确提示

### 3. 输入链路稳定

- 文字正常
- 语音正常
- 图片正常
- 图片预览正常

### 4. 页面可用

- 桌面端正常
- 手机端正常
- 关键交互无明显阻塞

### 5. 精算核对可开展

- 定价工具可运行
- 文档齐全
- 边界清楚
- 可移交精算部门
