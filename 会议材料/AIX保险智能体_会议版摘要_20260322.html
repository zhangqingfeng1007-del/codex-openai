AIX保险智能体会议版摘要

基于《AIX智能体需求与开发文档 v4.1》整理，供 2026-03-23 会议沟通使用

一句话目标：基于公司现有保险产品数据库、责任标准库和精算能力，建设一个以 AI 对话为核心的保险规划与推荐系统，让普通人成为保险专家。

1. 为什么现在做

  • 公司已经具备 1 万+ 结构化保险产品数据库、责任标准库和精算能力，当前最缺的是统一的智能体交互层和规划闭环。
  • 项目重点不是“从零造数据”，而是把既有数据资产、规则能力和 AI 交互能力连成产品。
  • 当前第一阶段优先服务个人用户，先把重疾险主链路做稳，再扩险种、扩用户、扩性能。

2. 第一阶段要交付什么

维度 目标
用户体验 首页极简，以“开始对话”为唯一焦点，欢迎态与对话态清晰分离。
输入能力 支持文字、语音、图片识别三种输入。
产品范围 聚焦重疾险（aix_category_id=6001）和全量在售产品。
结果能力 具备需求分析、预算测算、产品评分、精算报告能力。
终端 桌面网页和手机网页双端响应式可用。

3. 核心产品设计决策

3.1 首页定位

  • 首页不是流程导航页，而是 AI 对话入口。
  • 页面主体层级固定为：保险智能体 → 人人都是保险专家 → 有任何保险问题，尽管问我。
  • 左上角品牌统一为蓝色文字 AIx，不再使用厚重科技风或 PNG Logo。

3.2 两条规划路径

  • 当用户提出规划类需求时，先弹出二选一路由卡。
  • 填写信息规划：跳转已有 AIX 测算引擎 Module1，不新建独立规划页。
  • 对话规划：继续在 AI 对话里补充信息。
  • 两条路径共用同一套底层 schema，前端只做交互分流，不做业务分流。

4. 当前系统架构

层 职责
前端 React 保险智能体首页、对话页、AIX 测算引擎入口、结果卡片展示。
BFF Node.js 会话管理、客户档案、计算代理。
AI 服务 Python SSE 对话编排、系统提示词、需求分析、预算测算、产品评分、记忆系统。
精算服务 Java 费率计算和精算模型。

5. 本轮版本收口点

  • 品牌区改为蓝色 AIx 文字，首页视觉层级收口。
  • 旧 HomePage 降级为测算引擎内部入口，默认首页统一到 ChatPage。
  • 预算分配能力正式进入 store 接口，为后续 UI 展示留好结构。
  • 规划类需求统一走 route_options 路由卡，target 统一为 aix-engine。
  • 验收清单已经补全，便于按固定口径回归。

6. 当前会议建议重点

议题 会议里要确认的点
第一阶段边界 是否继续只聚焦重疾险主链路，不在本轮扩展到医疗险和 B 端。
交互闭环 是否确认首页以 AI 对话为唯一入口，结构化测算作为路由后的内部引擎。
数据与精算协同 产品标准库、责任库、费率引擎、预算测算规则的协同节奏如何安排。
验收口径 是否以品牌层级、路由逻辑、预算能力、结构化推荐结果作为本轮正式验收项。

7. 明天会议建议话术

  • 我们不是在做一个新的问卷工具，而是在把公司现有数据资产和精算能力产品化。
  • 第一阶段先把重疾险智能规划主链路做实，先证明“AI 对话 + 结构化推荐 + 精算支撑”这条路跑得通。
  • 架构上已经明确：ChatPage 是统一入口，AIX 测算引擎是被路由复用的内部能力，不再各自为战。
  • 接下来管理重点不是继续发散需求，而是按边界推进、按验收口径收口、按分工并行推进。
