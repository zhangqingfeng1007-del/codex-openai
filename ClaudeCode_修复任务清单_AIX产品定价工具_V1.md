## 3. P0 计算正确性问题

### P0-1 修复 CI 自定义模式的重复计入

**问题描述**

当前 `CI` 自定义模式下，既保留了“25种重疾（聚合）”，又保留了组成该聚合的逐病种列表。  
一旦进入自定义模式，代码会把聚合项和逐病种项同时累加，导致重疾发生率重复计入。

**问题位置**

- `insurance-product-tool.html`
- `CI_DISEASES`
- `computeCI()`
- `rawCIRate` 计算逻辑

**已确认现象**

以 35 岁男性为例：

- `CI_AGG.male[35] = 0.001396`
- 逐病种求和约为 `0.0006778`
- 两者叠加后约为 `0.0020738`
- 重疾脱退率被放大到约 `1.49x`

**修改要求**

请在自定义模式下，确保聚合项不再参与逐病种累加。可接受方案：

1. 切换到自定义模式时自动取消聚合项
2. 聚合项只允许在聚合模式存在，自定义模式不显示
3. 自定义模式内部忽略 `isAggregate=true` 的项

**验收标准**

1. 聚合模式下，CI 使用 `CI_AGG`
2. 自定义模式下，CI 只使用被勾选的逐病种
3. 自定义模式切换后，不会再出现“聚合项 + 逐病种”同时计入

---

### P0-2 修复 ToAge 只影响赔付、不影响脱退的问题

**问题描述**

当前赔付侧计算 `qCI` 时，会检查病种的 `ToAge`。  
但生存概率递减中的 `rawCIRate` 没有同步检查 `ToAge`。

这意味着：

- 某病种在保障年龄之后虽然不再赔付
- 但仍继续让合同脱退
- 会系统性抬高保费

**问题位置**

- `insurance-product-tool.html`
- `calculateCore()`
- `rawCIRate` 计算逻辑

**修改要求**

重疾病种进入脱退率计算时，必须和赔付侧保持同口径，至少同时满足：

1. 病种已选中
2. 当前年龄 `< ToAge`
3. 聚合/自定义模式正确切换

**验收标准**

1. 当某病种 `ToAge = 40` 时，40 岁及以后不能继续影响 `px`
2. 赔付逻辑与脱退逻辑使用相同的病种有效性判断

### P0-3 移除无效内联 onchange 处理器

**问题描述**

`buildDiseaseList()` 动态生成的 HTML 中包含：

```html
onchange="diseases[i]..."
```

但 `diseases` 只是函数局部参数，并不是全局变量。  
浏览器交互时会触发 `ReferenceError`。

虽然当前又额外绑定了 `addEventListener`，但界面仍然会报错。

**问题位置**

- `insurance-product-tool.html`
- `buildDiseaseList()`

**修改要求**

删除所有内联 `onchange`，只保留显式绑定的监听器。

**验收标准**

1. 病种勾选、赔付比例修改、`ToAge` 修改时，浏览器控制台无报错
2. 数据仍能正确更新

## 4. P1 产品开发工具能力缺口

### P1-1 支持责任级独立保额

**问题描述**

当前只有一个全局 `sumAssured`。  
这无法表达真实产品中：

- 身故保额与重疾保额不同
- 轻症/中症按总保额比例赔付
- 或轻症/中症为额外固定保额

**修改要求**

至少新增以下责任级配置能力：

1. `death_sum_assured`
2. `ci_sum_assured`
3. `minor_ci_benefit_mode`
4. `minor_ci_sum_assured_or_pct`
5. `moderate_ci_benefit_mode`
6. `moderate_ci_sum_assured_or_pct`
7. `special_ci_benefit_mode`
8. `special_ci_sum_assured_or_pct`

**建议支持的轻中症模式**

1. 占重疾保额百分比
2. 占基准总保额百分比
3. 额外固定保额

**验收标准**

1. 可以独立输入身故保额与重疾保额
2. 可以为轻症设置“30%总保额”或“额外10万元”两种模式
3. 公式中不再把所有责任都绑定到同一个 `sumAssured`

---

### P1-2 支持多缴费方式并行配置

**问题描述**

当前只有一个单值 `premPayPeriod`。  
这不符合产品开发工具场景，因为产品通常需要并行配置：

- 趸交
- 3年交
- 5年交
- 10年交
- 20年交
- 30年交

**修改要求**

将单个 `premPayPeriod` 改为缴费方案集合，例如：

```js
paymentOptions = [
  { code: 'single', payYears: 1, label: '趸交' },
  { code: '3y', payYears: 3, label: '3年交' },
  { code: '5y', payYears: 5, label: '5年交' },
  ...
]
```

**验收标准**

1. 页面可同时维护多个缴费方式
2. 试算与费率表生成能按不同缴费方式分别输出结果

---

### P1-3 支持“缴费方式 × 性别 × 投保年龄”的承保规则

**问题描述**

你们当前的真实需求是：

- 不同缴费方式
- 不同性别
- 不同投保年龄

可能对应不同的承保范围。

当前实现没有这种规则模型。

**修改要求**

每个缴费方案至少支持：

1. 适用性别
2. 最小投保年龄
3. 最大投保年龄

如果未来需要更细，也应预留：

4. 最小保障年龄
5. 最大保障年龄

**验收标准**

1. 可为趸交、3年交、5年交分别设置男女不同年龄区间
2. 不在区间内的组合不出费率结果

---

### P1-4 性别专属病种需要真正禁选

**问题描述**

当前页面只通过 `genderNote` 做提示，但没有真正禁用不适用病种。

例如：

- 男性仍可看到并勾选宫颈癌
- 女性仍可看到并勾选前列腺癌

虽然某些数据数组可能为 `0`，但这不是正确的产品工具行为。

**修改要求**

至少实现以下其一：

1. 不适用病种直接隐藏
2. 不适用病种显示但 `disabled`

**验收标准**

1. 男性不能勾选女性专属病种
2. 女性不能勾选男性专属病种
